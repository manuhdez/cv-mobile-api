# Get all users

The main functionality of the API is th get the full list of users stored in the database.

To get those users you need to to a get request to the url below.

Endpoint

https://cv-mobile-api.herokuapp.com/api/users

Fetch request example

fetch('https://cv-mobile-api.herokuapp.com/api/users')
  .then( response => response.json() )
  .then( jsonResponse => console.log(jsonResponse) )

Ajax request example

$.ajax('https://cv-mobile-api.herokuapp.com/api/users')
  .done( response => console.log(response))

# User pagination

Sometimes there is no need to fetch all users in one request, instead of it you may prefer to get just a group of users.

The main cases to use pagination instead of fetching all users are: Infinite scroll, lazy loading, etc.

To make it easier to develop those functionalities there is an endpoint where you will get as response an object with 10 users, as they appear on the database.

Endpoint

https://cv-mobile-api.herokuapp.com/api/users/pages/1

This url works similar to the previews one but adds a dynamic parameter at the end to render different user pages.

For every page you request, the api skips the needed users to avoid repetition so if you request for the 3rd page the response will be an object with the users 21 to 30.

Fetch request example

fetch('https://cv-mobile-api.herokuapp.com/api/users/pages/2')
  .then( response => response.json() )
  .then( jsonResponse => console.log(jsonResponse) )

Ajax request example

$.ajax('https://cv-mobile-api.herokuapp.com/api/users/pages/2')
  .done( response => console.log(response))

# Add a new user

To add a new user to the api database it is needed to make a POST request via AJAX.

The main point to be aware of is that you have to create a fomData object in order to send the mixed inputs info its needed to create a user profile.

Endpoint

https://cv-mobile-api.herokuapp.com/api/users

Creating a formData object.

function createRequestBody() {
  let formData = new FormData();

  // Sent as a string
  formData.append('name', name.value);

  // Sent as a string
  formData.append('username', username.value);

  // Sent as a string with email validation
  formData.append('email', email.value);

  // Sent as a string
  formData.append('city', city.value);

  // Sent as a string
  formData.append('country', country.value);

  // Sent as a string
  formData.append('jobTitle', jobTitle.value);

  // Sent as a string
  formData.append('website', website.value);

  // Sent as a string
  formData.append('company', company.value);

  // Sent as a string. Choose a value from below.
  formData.append('experience', experience.value);

  // Pick the value from an input(type="date")
  formData.append('birthDate', birthDate.value);

  // Pick the value from an input(type="file") that accepts only jpeg and png formats and files under 3MB size
  formData.append('profilePicture', profilePicture.files[0]);

  // Store all the values selected in the form inside an Array and parse it as a string
  formData.append('languages', JSON.stringify(languagesArray));

  // Store all the values selected in the form inside an Array and parse it as a string
  formData.append('skills', JSON.stringify(skillsArray));

  return formData;
  }

Post request examples

function sendNewUser(e) {
  e.preventDefault();
  let formBody = createRequestBody();

  fetch('https://cv-mobile-api.herokuapp.com/api/users', {
    method: 'POST',
    body: formBody
    })
    .then( res => res.json())
    .then( response => console.log(response));
  }

# Update data of an existing user

To update the info of a user inside the database it is needed to make a PUT request via AJAX with the user's id.

It works like the add a new user endpoint, so you have to be again aware to create a fomData object in order to send mixed inputs data.

To avoid changing every users data when sending the request be sure that you send every data the user already have stored or it will be erased after the request.

To do that its recomended to fetch the users info first and then sending the same info but with the values changed where you need it.

Endpoint example

https://cv-mobile-api.herokuapp.com/api/users/5bc0af41aa11020015337942

# Delete an existing user

To delete a user from the database you just need to now its _id property.

Then just create a DELETE request as explained below.

Endpoint example
